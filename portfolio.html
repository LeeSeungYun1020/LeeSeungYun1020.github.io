러닝 트립

2022년 6월 ~ 개발 중

https://github.com/SWM-TRAVELY/LearningTrip

아이와 여행을 떠나는 부모에게 관광지 정보를 제공하고 지역 전문 해설사와 탐방 가능한 플랫폼

설명

  • 아이와 함께 교육적인 여행 떠날 때 아이에게 지식을 잘 전달하고 싶은 부모들의 고민 해결
  • 부모는 아이의 지적 수준 향상시키고 수행 평가 과제를 위해 교육적인 여행의 필요성 커짐
  • 전문 지식이 부족한 부모들은 관광지 정보를 확인하고 해설사를 선택하여 아이와 함께 탐방에 참가할 수 있음

주요 기능

관광지 정보와 리뷰, 키워드로 관광지 탐색, 탐방(해설사와 매칭), 여행 스케줄 생성, 스티커 및 레벨과 업적

담당

Android

사용 기술

Kotlin, Jetpack (compose, lifecycle, navigation, room), Coil

  • 어플리케이션 상태 변경에 따라 UI를 재생성하는 Jetpack Compose 사용
    • 적은 코드로 UI를 구성하고 코틀린으로 빠르게 개발할 수 있다고 판단하여 도입
    • 어플리케이션 상태가 바뀔 때마다 UI 업데이트가 필요한데 뷰를 사용할 경우 뷰마다 상태가 다르고 각각 업데이트해야 해 과정이 복잡하고 빼먹는 등 에러가 발생한 경험이 있었음
  • 오프라인 환경에서도 관광지 정보를 확인할 수 있도록 Room을 이용하여 로컬 데이터베이스를 구성하여 관광지 정보를 표시하고 검색하는 기능을 구현
    • SQLite에 비해 쉽고 편리하게 사용할 수 있어 Room을 선택, Room은 컴파일 시간에 SQL 쿼리가 유효한지 확인해 주며 상용구 코드 없이 레코드를 객체로 변환시켜주는 장점이 있음
    • 보완하면 좋은 점으로 초기 데이터 입력을 위해 assets의 csv파일이 15mb 정도의 용량을 차지함
  • compose를 지원하는 이미지 라이브러리 Coil을 사용하여 비동기적으로 이미지 요청을 보내고 표시하도록 구현함
  • navigation 라이브러리를 통해 원하는 screen을 바로 표시할 수 있도록 구현
    • navigation을 이용하지 않으면 별도의 상태 변수와 이전 스크린 stack을 저장해야 하는 불편함이 있음
  • MVVM 패턴을 적용
    • compose로 view 관련 코드를 모두 분리해서 작성할 수 있고 네트워크와 데이터베이스 관련 코드는 viewmodel로 처리하며 데이터는 model로 처리하여 읽기 쉽고 유지보수가 편한 코드를 작성할 수 있다고 판단함
  • 지연 목록(LazyColumn, LazyRow)을 사용하여 데이터베이스 및 네트워크에 영향을 받는 목록을 관리하여 io를 줄임
  • 스크럼을 도입하여 매일 일일 스크럼 회의를 통해 개발 진행사항과 잘 안되는 점, 당일 할 일을 공유하였고 매주 토요일 스프린트 계획 회의를 통해 다음 스프린트 계획을 세움

GETs

2021년 4월 ~ 2022년 2월

https://github.com/LeeSeungYun1020/Gets

맞춤 추천과 가상 피팅으로 의류 선택 고민을 해결해주는 개인 맞춤형 패션 플랫폼

설명

  • 매일 입는 옷, “오늘 뭐 입지?” 라는 고민 해결
  • 기존 코디 앱은 코디 데이터를 보고 사용자가 직접 코디 방법을 학습해야 했음
  • 사용자의 체형, 좋아요, 성별, 나이, 계절에 따른 맞춤 추천 코디를 제공하여 사용자 경험 개선
  • 직접 입어 보기 전에 확인할 수 있도록 가상 피팅 기능을 통해 스타일링해볼 수 있도록 함

주요 기능

맞춤 추천, 가상 피팅, 스타일 가이드, 탑 트렌드, 제품(탐색, 상세), 옷장(제품, 코디)

담당

Android, Server, Unity

사용 기술

Kotlin, Jetpack (viewBinding, lifecycle, navigation), Volley, JavaScript, Node.js, Express, MySQL, C#, Unity, EJS, SCSS

  • viewBinding을 이용하여 findViewBy 없이 레이아웃과 상호작용하는 코드를 쉽게 작성
    • 기존에 사용했었던 kotlin-android-extensions를 대체하기 위해 viewBinding을 사용하게 됨
  • BottomNavigation을 이용하여 주요 기능을 수행하는 홈, 제품, 옷장, 계정 fragment간에 이동 가능하도록 구현
  • Volley 라이브러리를 사용하여 네트워크 통신을 수행. json 요청을 보내고 json 데이터와 이미지를 받아와 표시함
    • 서버 API는 앱뿐만 아니라 다른 팀원이 개발하는 웹에서도 동일한 API를 사용하기 때문에 API 문서를 직접 작성하고 관리함.
  • RecyclerView를 사용하여 네트워크에서 받아오는 많은 데이터를 효과적으로 표시
  • MVC 패턴을 적용
  • 매주 금요일 회의를 진행하여 개발 진행 사항을 확인하고 다음 개발 방향을 의논함

2020년 4월 ~ 2020년 7월

중요도와 긴급도에 따라 우선 순위 기반 할 일 목록을 추천해주는 시간 관리 앱

설명

  • 시간 관리 고민을 해결하기 위한 시간 관리 방법을 제시하는 앱
  • 우선 순위 기반 할 일 목록을 제공하고, 프로젝트 별로 할 일을 분류하며, 중요도-긴급도 행렬을 볼 수 있으며, 목표 달성 정도를 분석할 수 있음
  • 앱을 통해 시간 관리 부담을 낮추고 시간 낭비를 줄임, 시간 관리 방법을 통해 똑똑하고 효율적으로 원하는 목표를 달성할 수 있도록 함

주요 기능

우선 순위 기반 할 일 목록, 프로젝트별 할 일 분류, 중요도-긴급도 행렬, 목표 달성 정도 분석

담당

Android, Server

사용 기술

Kotlin, Jetpack (lifecycle, navigation, room), Volley, JavaScript, Node.js, Express, MySQL

  • Navigation Drawer를 사용하여 할 일, 프로젝트, 행렬, 분석, 설정 fragment 간에 이동할 수 있도록 구현
  • Room 로컬 데이터베이스를 이용하여 할 일과 프로젝트 데이터를 관리
  • Volley 라이브러리를 이용하여 동기화 서버와 통신하여 할 일, 프로젝트 데이터를 서버에 동기화할 수 있는 기능을 구현
  • 디자인 가이드를 적용하기 위해 Material UI 선택, 앱의 테마를 일관성 있게 관리
  • 데이터베이스 관련 코드 작성에서 유지보수를 쉽게 하기 위해 MVVM 패턴을 적용

바름

2022년 1월 ~ 2022년 6월

https://github.com/LeeSeungYun1020/Baleum

QR코드로 수강한 교육 과정을 한 번에 증명할 수 있는 블록체인 기반 학습 인증 서비스

설명

  • 매번 재발급이 필요하고 위변조 확인이 필요한 인증서, 강의명만 나와있는 인증서, 처음 듣는 곳에서 발급한 인증서 같이 기존 학습 인증서의 문제점을 해결
  • 학습자는 강의 증명서 재발급이나 누락 없이 QR코드 하나만 제출하여 학습 이력을 관리할 수 있음
  • 검증자는 위변조 걱정 없이 학습자가 배운 과정과 수준을 정확히, 한 번에 파악할 수 있음

주요 기능

강의 수강, 강의 생성, 블록체인을 통한 학습 진행 과정 저장, 블록체인을 통한 수업 수료 인증, QR코드로 수료 확인(인증서)

담당

Server, Blockchain

사용 기술

JavaScript, Node.js, Express, MySQL, Solidity, Truffle, Web3

  • Solidity로 스마트 컨트랙트를 작성하고 Truffle로 컴파일 및 마이그레이션하여 이더리움 네트워크 상에서 dApp을 구동
    • Truffle은 dApp을 빠르게 구성할 수 있도록 여러 템플릿을 구성합니다. 이 중 Express-Box를 참고하여 API 서버를 빠르게 제작할 수 있었음
  • 로컬 블록체인인 가나슈를 사용하여 스마트 컨트랙트를 배포하고 테스트
    • 이더리움 네트워크에 직접 배포해보지 못한 점이 아쉬움. 비싼 이더리움 대신 코드가 호환되는 클레이튼, 트론을 통해서 직접 배포해보았다면 좋았을 것
  • MVC 패턴을 적용하여 router와 controller로 분리하여 구현
    • 사용자(react)는 router(view)를 통해 API를 요청하고 controller가 데이터베이스 및 블록체인 네트워크에 접근하여 데이터를 전달할 수 있도록 MVC 패턴을 사용
  • web3와 truffle을 이용하여 서버에서 블록체인 네트워크와 상호작용
  • 매주 화요일 운죽정에서 만나 개발 상황을 점검하고 다음 스프린트 계획을 세움
